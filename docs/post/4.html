注解和反射

1 注解 Annotation

场景类比：如果将程序看作商品，注解就是特意贴在商品上面的小标签，比如超市促销打八折。也就是说他和商品本身没什么关系，只是需要理货员需要把它放到特价区之类的，顾客之类的。

1.1 官方注解

1.1.1 内置注解

内置注解就是java官方提供的3个直接可使用的注解，可以看到都是处理编译问题，帮忙检查的注解。

@Override 重写注解

@Deprecated  不推荐使用该方法，一般用在在即将弃用的老方法上

@SuppressWarnings("all")  如果方法没有被调用，会灰色并提示，这个注解会取消这个提示

1.1.2 元注解

用来解释注解的注解。只能作用于注解上。

@Target()  空间，注解可以用在什么地方，方法上？类上？

@Retention()  时间，啥时候还有用？三阶段，SOURCE < CLASS < RUNTIME。自定义注解一般都需要在runtime还有效

@Documented 表示是否将我们的注解生成在javadoc中

@Inherited 子类可以继承父类的注解

2.2 自定义注解

@interface 自动继承java.lang.annotation.Annotation
直接以示例展示注解的写法


//存在默认值设置可以不传参则是默认值，如果没有default，则必须传。
@MyAnnotation(name = "Holmes",schools = {"University of Cambridge"})
public void test(){}

@Target({ElementType.TYPE,ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnotation{
    String name();
    int age() default 0;
    int id() default -1;
    
    String[] schools();
}

// 但一般不会定义这么复杂的注解，如果值只有一个可以用value，这样可以省略 value。不然得键值对应。


@MyAnnotation2("Holmes")
public void test(){}


@Taget({ElemenType.TYPE,ElementType.METHOD})
@Retention(RetentionPolicy.RUNTIME)
@interface MyAnnotation2{
    value();
}

2 反射 Reflection

经典动态语言python可以随意操控，自由度极高，但牺牲了效率。需要编译的静态语言java也想像动态语言一样运行时操控程序，修改类型，怎么办呢。
于是反射技术应运而生，也就是静态语言java的动态技术。

# 变量a可以随时改类型，无编译检查 
a = 10 # 整数 
a = "hello" # 字符串（无报错） 
a = [1,2,3] # 列表（无报错） # 错误只有运行时才暴露 
def add(x, y): 
    return x + y add(10, "20") # 运行时才抛出TypeError，编译阶段完全无提示

  1. Class 对象本质是 Java 中「类型」的抽象表示，只要是能作为「类型」声明变量 / 参数的，都有 Class 对象。
  2. interface 接口
  3. [] 数组
  4. enum 枚举
  5. annotation 注解@interface
  6. primitive type 基本数据类型
  7. void

  6. 包java.lang、com.example

2.3 java.lang.reflect 操作Class对象

2.3.1 Class类

这和2.1的获得Class不要搞混了，那个相当于"new"出class，这个才是对其的操作 !!!

  • 功能：表示类的对象，提供了获取类信息的方法，如字段、方法、构造函数等。
  • 主要方法：
    • getFields()：获取所有公共字段。
    • getDeclaredFields()：获取所有声明的字段，包括私有字段。
    • getDeclaredField() : 单个获取，传参拿指定的方法，其他方法差不多，仅演示一次这种写法。
    • getMethods()：获取所有公共方法。
    • getDeclaredMethods()：获取所有声明的方法，包括私有方法。
    • getConstructors()：获取所有公共构造函数。
    • getDeclaredConstructors()：获取所有声明的构造函数，包括私有构造函数。
    • getSuperclass()：获取类的父类。
    • getInterfaces()：获取类实现的所有接口。

显而易见的get了很多不同的东西，那你的用对应的东西接啊，于是就java.lang.reflect也应该提供对应的Field 类 ， Method 类，Constructor 类和其操作

2.3.2 Field 类

  • 功能：表示类的字段（属性），提供了访问和修改字段值的方法。
  • 主要方法：
    • get(Object obj)：获取指定对象的字段值。
    • set(Object obj, Object value)：设置指定对象的字段值。
    • getType()：获取字段的数据类型。
    • getModifiers()：获取字段的修饰符（如 public、private）。

 2.3.3 Method 类

  • 功能：表示类的方法，提供了调用方法的能力。
  • 主要方法：
    • invoke(Object obj, Object... args)：调用指定对象的方法。
    • getReturnType()：获取方法的返回类型。
    • getParameterTypes()：获取方法的参数类型。
    • getModifiers()：获取方法的修饰符（如 public、private）。

2.3.4 Constructor 类

  • 功能：表示类的构造函数，提供了创建对象的能力。
  • 主要方法：
    • newInstance(Object... initargs)：创建一个新实例，使用指定的构造函数参数。
    • getParameterTypes()：获取构造函数的参数类型。
    • getModifiers()：获取构造函数的修饰符（如 public、private）。

2.3.5 拓展

这篇文章只介绍基础用法，实际上为了支持泛型，method还有getGenericParameterTypes()方法专门拿泛型，并且有ParameterizedType类可以用来处理泛型。

3 反射实现注解

注意 Annotation是 java.lang.annotation包提供的

// 获取其注解
Annotation[] annotations = class111.getAnnoctations();

// 获取其值
annotation.value();

// 那么有聪明的老铁就要问了，那注解写在字段上咋办呢

// 1.用反射的方法拿到该字段。   假设class1是类，name是那个字段名。
Field f = class1.getDeclaredField("name");
// 2.该字段作为一个类又能就能进行和普通class一样的注解操作啦。   假设Fieldtest是name的字段注解名。
Fieldtest annotation = f.getAnnotation(Fieldtest.class);
System.out.println(annotation.value());//拿到注解值啦

这个示例只是初现端倪，我为了示例的简单，直接指定了，实际上Fiedtest，name，这些东西都不用你事先知道，你也可以用反射获取他们，然后利用反射得到的东西继续操作，还可以根据反射得到的东西的得到的东西。。。。
